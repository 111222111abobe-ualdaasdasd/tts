# TTS Студия — синтез речи на базе Microsoft Edge

Бесплатный онлайн-синтез речи: сотни нейронных голосов на десятках языков,
настройка скорости / тона / громкости, история и скачивание MP3.

Работает через сервис Microsoft Edge «Read Aloud» (без API-ключей).

## Локальный запуск

```bash
npm install
npx drizzle-kit push      # создать таблицы в локальной PostgreSQL
npm run dev               # http://localhost:3000
```

Нужна переменная окружения `DATABASE_URL` в файле `.env`.

## Деплой на Timeweb Cloud (бесплатно, из РФ без VPN)

Разворачивается одной командой через Docker Compose: поднимется и приложение,
и база PostgreSQL на одном сервере.

### 1. Создай сервер
1. Зарегистрируйся на [timeweb.cloud](https://timeweb.cloud).
2. **Облачные серверы → Создать**.
3. Образ: **Ubuntu 22.04 / 24.04**.
4. Тариф: минимальный (1 ядро, 1–2 ГБ RAM) — для старта хватает. Можно
   использовать бесплатный промо-тариф, если доступен.
5. Создай сервер и скопируй его **IP-адрес** и пароль root.

### 2. Подключись по SSH
На своём компьютере (в терминале):
```bash
ssh root@IP-АДРЕС
```

### 3. Установи Docker одной командой
```bash
curl -fsSL https://get.docker.com | sh
```

### 4. Склонируй проект и запусти
```bash
git clone https://github.com/ТВОЙ_ПОЛЬЗОВАТЕЛЬ/ТВОЙ_РЕПО.git tts
cd tts
docker compose up -d --build
```
Сборка займёт ~2–4 минуты. По окончании сайт будет доступен по адресу:
```
http://IP-АДРЕС
```

### 5. (Опционально) Привязать домен
- В настройках DNS твоего домена создай **A-запись** → на IP сервера.
- В панели Timeweb можно бесплатно получить SSL-сертификат или
  настроить nginx. Но и по `http://IP` всё работает сразу.

## Где что хранится
- **База данных** (история + аудио): в Docker-томе `tts_pgdata` — не теряется
  при перезапусках. История автоматически ограничена последними 40 записями.
- **Код**: в твоём Git-репозитории.

## Деплой на Netlify (serverless)

Netlify запускает Next.js API-роуты как serverless-функции (AWS Lambda).
Для этого проекта есть два важных нюанса (см. ниже «Риски»).

### 1. База данных (внешняя)
Netlify не предоставляет PostgreSQL, а Neon в РФ заблокирован. Используй
бесплатный PostgreSQL, доступный из РФ:
- **Supabase** (supabase.com) — 500 МБ, проще всего.
- **ElephantSQL** (elephantsql.com) — бесплатный план (20 МБ; для истории
  из ~40 записей по 30 КБ = ~1.2 МБ хватает).

Создай базу → скопируй connection string.

### 2. Примени схему к этой базе (локально):
```bash
DATABASE_URL="postgresql://...твоя-строка..." npx drizzle-kit push --force
```

### 3. Залей код на GitHub (`.env` в `.gitignore`!)

### 4. Импортируй в Netlify
- app.netlify.com → Add new site → Import → выбери репозиторий.
- Фреймворк Next.js определится автоматически.
- **Environment variables** → добавь `DATABASE_URL` = строка подключения.
- Deploy → получишь `https://<имя>.netlify.app`.

### Риски Netlify (важно!)
1. **Таймаут функции — 10 секунд** на бесплатном тарифе. Синтез речи занимает
   обычно 3–6 сек, но на длинном тексте может не уложиться → ошибка. На
   платном тарифе (26 сек) проблема уходит.
2. **Исходящий WebSocket к Microsoft** — технически Lambda разрешает исходящие
   соединения, и библиотека `ws` работает, но это не гарантируется. Если
   «Озвучить» не работает на Netlify — это, скорее всего, причина.
3. **Холодный старт** — первая генерация после простоя может занять дольше.

Если эти ограничения мешают — используй деплой на VPS/Timeweb (см. выше):
там долгоживущий сервер без таймаутов и с гарантированным WebSocket.

## Как это работает
- `src/lib/edge-tts.ts` — собственный клиент протокола Microsoft Edge TTS
  (WebSocket), с актуальной версией Chromium-идентификатора и muid-токеном.
- `src/app/api/synthesize` — генерирует речь и сохраняет MP3 в БД.
- `src/app/api/voices` — список голосов (берётся с серверов Microsoft).
